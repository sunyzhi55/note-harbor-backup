## 🔍 各部分说明

**01.AcWing基础课学习笔记**

用于整理 AcWing 基础课课程、题目模板和常见算法专题，适合按章节逐步沉淀笔记。

- **核心内容**：基础语法、常用数据结构、基础算法模板。
- **适用场景**：打牢基础，快速查阅标准代码实现。

**05.左程云学习笔记**

用于整理左程云课程中的算法思想、经典模型和题型总结，更偏向系统化进阶整理。

- **核心内容**：复杂算法思想、经典模型推导、面试高频题型。
- **适用场景**：提升解题思维，掌握算法背后的数学与逻辑之美。

**06.杂笔记**

用于放置暂时不适合单独成体系的算法碎片、题目感悟和临时总结。

- **核心内容**：周赛/竞赛复盘、单一题目的深度感悟、未分类的知识点。
- **适用场景**：捕捉瞬间的灵感，防止遗忘。

## 💡记录建议

为了保持笔记的高质量与可读性，建议遵循以下原则：

- **聚焦主题**：每篇笔记尽量只聚焦一个核心知识点或一类题型，避免大杂烩。
- **AcWing 类内容**：建议保留 **题意描述**  **解题思路**  **核心代码模板**  **时间/空间复杂度分析**。
- **左程云类内容**：建议重点补充 **模型归纳** 和 **适用场景**，多问自己“为什么这么做”。
- **零散记录**：可以先快速写下结论或关键点，后续有时间再进行系统性补充整理。



## 📋 目录使用原则

1. **优先归档**：新算法内容优先归入对应的子目录，保持根目录整洁。
2. **命名规范**：章节笔记（理论）和题解（实践）尽量分开命名，方便检索。
3. **动态调整**：如果某个杂项内容逐渐成熟，再考虑将其拆分成更细的专题目录。
